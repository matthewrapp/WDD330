  • Object Literals
    • A self-contained set of related values and functions
    • Act as a collection of named properties that map to any JavaScript value such as:
      • Strings
      • Numbers
      • Booleans
      • Arrays
      • Functions
    • If a properties value is a function, it is known as a method
    • A database of values
    • Objects are often used to keep any related information and functionality together in the same place. For Example:
      • If you wrote functions that found the perimeter and area of a square, you might want to group them together as methods of the same object that also included a length property.
    • Super Example:

      • const superman = {
        name: 'Superman',
        'real name': 'Clark Kent',
        height: 75,
        weight: 235,
        hero: true,
        villain: false,
        allies: ['Batman', 'Supergirl', 'Superboy'],
        fly() {
        return 'Up, up and away!';
        }
        }
      • Each property is a key-value pair.
      • 'fly()' property is a method
      • If property names doesn't follow the rules for naming variables described in ch. 2, it needs to be quoted.
    • Creating Objects
      • To creeate an object literal, simple enter a paid of curly braces. Example: '{}'
      • I'ts also possible to create an object using a constructor function. This example will also create an empty object.
        • const spiderman = new Object();
        • This method is not recommended, however, and the object literal notation is the preferred way of creating objects.

      • const name = 'Iron Man';
        const realName = 'Tony Stark';

        // long way
        const ironMan = { name: name, realName: realName};

        // ES6 way
        const ironMan = {name, realName};
    • Accessing Properties
      • Using dot notation:
        • superman.name
          Output: 'Superman'
      • Using bracket notation:
        • superman['name']
          Output: 'Superman'
      • Dot notation is more common, but bracket notation is the only way to access nonstandard property and method names that don't follow variable naming rules.
      • Example:
        • superman["real" + " " + "name"]
          Output: "Clark Kent"
    • Computed Properties
      • The ability to create objects with computed property keys was introduced in ES6. This means the Javascript code can be placed inside square brackets and the property key will return the value of that code.
      • Example:

        • const hulk = {
          name: 'Hulk',
          ['catch' + 'Phrase']: 'Hulk Smash!'
          }

  • What is the DOM?
    • Document Object Model
    • Represents an HTML document as a network of connected nodes the form to a tree-like structure.
    • The DOM treats everything on a web page as a node.
    • The HTML tag is the root node, and every other part of the document is a child node of this.
  • Getting elements
    • The DOM provides several methods that allow us to access any element on a page. These methods will return a node object or a node list, which is like an array-like object. These objects can then be assigned to a variabel and be inspected or modified.
    • Example: Accessing the body element of a webpage
      • const body = document.body;
    • We can use the nodeType property to find out what type of node it is. Example:
      • body.nodeType;
        Outputs: 1
    • All nodes have a numerical code to signify what type they are.
      • Code | Type
      • 1 | element
      • 2 | attribute
      • 3 | text
      • 8 | comment
      • 9 | body
    • We can use the nodeName property to find the name of the elemen. Example:
      • body.nodeName;
        Outputs: "BODY"
  • Legacy DOM Shortcut Methods:
    • document.body
    • document.images

      • Returns a node list of all the "a" elements and "area" elements that hafe an "href" attribute
    • document.anchors
      • returns a node list of all the "a" elements that have a "name" attribute
    • document.forms

    • Powerful way of specifying very precise items on a page. Example:
      • const wonderWoman = document.querySelector('li:last-child');
    • jQuery
      • A popular JavaScript framework that makes it very easy to find elements on a page ussing a CSS-style syntax.
      • It uses document.querySelectorAll() in the background whenever it can.
      • Example:
        • $('ul#roster').find('li#bats');
      • Same as doing:
        • const ul = document.querySelector('ul#roster');
          ul.querySelector('li#bats')

  • Event Listeners
    • Click Event
      • Occurs when a user clicks with the mouse, presses the Enter key, or taps the screen, making it a very useful all-around event covering many types of interaction.
    • The following code can be used to attach an event listener to the document the fires when the user clicks anywhere on the page:
      • document.body.addEventListener("click", doSomething);
  • In-line Event Handlers
    • The original way of dealing with events in the browswer was to use inline attributes that were added directly into the markup.
    • Example:
      • p onclick = "console.log('You clicked me')" | Click Me | p
    • For these reasons, avoid inline event handlers:
      • The JavaScript code is mixed up with the HTML markup, breaking the concept of unobtrusive JavaScript, which advocates that JavaScript code should be kept out of the HTML.
      • Only one event handler for each event-type can be attached to an element.
      • The code for the event handlers is hidden away in the markup, making it difficult to find where these events have been declared.
      • The JavaScript code has to be entered in a string, so you need to be careful when using apostrophes and quote marks.
  • Using Event Listeners
    • The recommended way of dealing with events, and the current standard, is to use event listeners.
    • These were outlined in DOM level 2 and allow multiple functions to be attached independently to different events.
    • The addEventListener() method is called on a node object, the node to which the event listener is being applied.
    • Example:
      • document.body.addEventListener('click', doSomething);
    • The addEventListener() method can also be called without a node, in which case it is applied to the global object, usually the whole browser window.
    • Its first parameter is the type of event, and the second is a callback function that is invoked when the event occurs.
    • Now, we are adding a click event listeer to the whole page. Two ways to write it, anonymous function adn declartion function:
      • addEventListener('click', () => alert('You Clicked!'));
      • function doSomething()
        { alert('You Clicked!');
        }
        addEventListener('click',doSomething);
  • The Event Object
    • Whenever an event handler is triggered by an event, the callback function is called.
    • This function is automatically passed an event object as a parameter that contain information about the event.
    • Example:
      • function doSomething(event) {
        console.log(event.type);
        }

        >> click
  • Coordinates of an Event
    • There are a variety of ways to find the position of where a mouse event occurs.
    • The screenX and screenY properties show the number of pixels from the left and top of the screen respectively where the event took place.
    • The clientX and clientY properties show the number of pixels from the left and top of the client that is being used (usually the browswer window).
    • The pageX and pageY properties show the number of pixels from the left and top, respectively, where the event took place in the document.
      • This property takes account of whether the page has been scrolled.
    • All these properties are similar, but subtly different.
    • They are useful for finding out the place where a click happened or the position of the mouse cursor.
  • Coordinates of an Event
    • Here's a link to all the types:
      • https://developer.mozilla.org/en-US/docs/Web/Events
  • Mouse Events
    • click
    • mousedown
      • Occur before the mouse event 'click' is fired.
    • mouseup
      • Occur before the mouse event 'click' is fired.
    • dblclick
      • Occurs when the user doubleclicks on the element
  • Keyboard Events
    • keydown
      • Occurs when a key is pressed and will continue to occur if the key is held down.
    • keypress
      • Occurs after a keydown event but before a keyup event.
    • keyup
      • Occurs when a key is released.
  • Touch Events
    • touchstart
      • Occurs when a user initially touches the surface.
    • touchend
      • Occurs when a user stops touching the surface.
    • touchmove
      • Occurs after a user has touched the screen then moves around without leaving.
    • touchenter
      • Occurs when a user has already starting touching the surface, but then passes over the element to which the event listener is attached.
    • touchleave
      • Occurs when the user is still touching the surface, but leave the element to which the event listener is attached.
    • touchcancel
      • Occurs when a touch event is interupted, such as a user's finger moving outside the document window, or too many fingers being used at once.
    • Has a length property that tells you how many touch points (usually the user's fingers) are in contact with the surface.
  • Removing Event Listeners
    • removeEventListener()
  • Stopping Default Behavior
    • preventDefault()
  • Event Propagation
    • The order that the events fire on each element
    • Two forms of propagation:
      • bubbling
        • When the event fires on the element click on first, then bubbles up the document tree, firing an event on each parent element until it reaches the root node.
        • Default behavior
        • Stopping the bubbling phase
          • Add the event.stopPropagation() method into the callback function.
          • Example:
            • liElement.addEventListener('click', (event) => {
              concole.log('clicked on li');
              event.stopPropagation();

              }, false);
      • capturing
        • Starts by firing an event on the root element, then propagates downwards, firing an event on each child element until it reaches the target element that was clicked on.
        • The addEventListener() method has a third parameter, which is a boolean value taht specifies whether captuing should be used or not. It defaults to false, which is why bubbling happens by default.
  • Event Delegation
    • Can be used to attach an event listener to a parent element in order to capture events that are triggered by it's child elements.

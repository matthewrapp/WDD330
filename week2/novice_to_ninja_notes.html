Chapter 2 Notes

  • Comments:
    • Comments are made starting with "//" on a single line.
    • Longer comments are made starting with "/*" & ending with "*/". Multi-line comment.
  • Syntax used by JavaScript is known as a C-style syntax because of its similarities with the "C" programming language.
  • Statement:
    • JavaScript is made up of a series of statements. Each statement ends with a new line or semicolon.
    • Example of a statement:
      • const message = 'Hello World!'
        alert(message)
  • Block:
    • A block is series of statements that are collected together inside curly braces.
    • Example of a block:
      • {
        const message = 'Hello!';
        alert(message);
        }
    • Do NOT need to be terminated by a semicolon.
  • Reserved Words:
    • Words reserved for use by the JavaScript language and CANNOT be used to name variables, functions, & object properties.
  • Primitive Data Types:
    • JS has seven different data types. Six of them are primitive data types:
      • String
      • Symbol (only introduced in ES6
      • Number
      • Boolean
      • Undefined
      • Null
    • Any value that isn't one of the primitve data types listed above is an object. These include arrays, functions, and object literals.
    • JS has a special operator called "typeof" for finding out the type of a value.
  • Declaring & Assigning Variables:
    • Keywords: "const", "let", & "var" declare variables.
    • You can declare multiple varibales at the same time by seperating them with commas. For Example:
      • "let x = 3, y = 5, z = 5"
    • Variables declared by "let" can be reassigned to another value at some point later in the program. For Example:
      • "let name = 'Matthew'; "
        " name = 'Jack'; "
    • Variables declared by "const" can NOT be reassigned to another value. For Example:
      • "const name = 'Matthew'; "
        " name = 'Jack'; " (This will bring up an error.
    • "var" isn't used much anymore due to the similarity of "let"
  • Scope
    • Scope:
      • Using "const" & "let" to declare variables means they are block scoped, so their value only exists inside the block they are declared in.
      • Scope referes to where a constant or variable is accessible by the program.
      • Two types of Scope, Global & Local.
    • Global Scope:
      • Any variable declared outside of a block is said to have a global scope. Meaning it is accessible everywhere in the program.
      • Might sound nice, but isn't considered good practice. Programmers try to keep the number of global variables to a minimum. Why?
        • Variables that share the same name will clash and potentially overwrite each other's values.
        • Very common to clash/overwrite other variables written.
    • Local Scope:
      • In ES6, blcoks can be used to create a local scope. Meaning that any variables defined inside a block using "let" or "const" will only be available inside that block.
  • Naming Constants and Variables
    • When naming constants and variables, you should try to give them sensible names that describe what the variable represents; hence, "answer" is a better variable name than "x".
    • Can start with any upper or lower-case letter, an underscore, or dollar character. They can also contain numbers, but can NOT start with them.
    • Examples:
      • $name
      • _answer
      • firstName
      • last_name
    • Variable Naming:
      • Variables that start with an underscore generally refer to private properties and methods, so it's best to not follow this convention for your own variable names.
      • The dollar character is also used bt the popular jQuery library, so using this in your variable names may also cause problems.
      • Constant and variable names ARE case sensitive.
      • When using multiple words for constant and variable names there are two conventions:
        • Camel Case (Probably the best convention to use), Example:
          • " firstNameAndLastName "
        • Underscore, Example:
          • " first_name_and_last_name "
  • Direct Assignement & Assignment by Reference
    • Direct Assignment
      • Example:
        const a = 1;
        let b = a; // a = 1, b = 1
        b = 2; // a = 1, b = 2
      • a references the primitive number 1 . We then assign b to the same value as a . At this point a and b both have the same value of 1 . Then we reassign the value of 2 to b , but the a still has a value of 1 .
    • Assignment by Reference
      • Example:
        const c = { value: 1 };
        let d = c; // c.value = 1, d.value = 1
        d.value = 2; // c.value = 2, d.value = 2
      • the change to the value property of d also results in the value property of c changing as well. This is because the variables c and d are both referencing thesameobject, so any changes to one of them will also affect the other.
  • Backslashes
    • Used to escape special characters in strings, such as:
      • Single quote marks: \'
      • Double quote marks: \"
      • End of the line \n
      • Carriage return \r
      • Tab \t
      • If you want to actually write a back slash:
        • "This is a blackslash \\"
  • Template Literals
    • Special types of string that were introduced in ES6.
    • Thought of as super-powered strings.
    • Use " ` " to deliminate the string
    • Used for this type of example, for a string:
      • `She said, "It's Me!"`
    • They also allow interpolation of JS code. Meaning that a JS expression can be inserted inside a string.
      • Example:
        const name = `Siri`;
        `Hello ${ name }!`;
  • Symbols
    • Introduced as a new primitive value in ES6.
    • Can be used to create unique values, which helps to avoid any naming collisions.
    • Only way to create them is to use the "Symbol()" function.
      • Example:
        const uniqueID = Symbol();
    • Recommended to add a description of the symbol inside perenthesis.
      • Example:
        const uniqueID = Symbol('this is a unique ID');
    • Can manually access the description using the String() function:
      • Example:
        String(uniqueID) = Symbol('this is a unique ID');
  • Octal & Hexadecimal Numbers
    • If a number starts with "0x", it is considered to be in hexademical (base 16) notation:
      • Example:
        0xAF; // A represents 10, F represents 15
        Output: 175
    • Hexademical or "hex" numbers are used used for color codes on the Web.
    • ES6 now supports octal literals: If a number starts with a zero, followed by the letter o, then it is considered to be in octal (base 8) notation:
      • Example:
        0o47; // 4 eights & 7 units
        Output: 39
    • ES6 also supports binary literals: If a number starts with a zero, followed by the letter b, then it is considered to be in binary (base 2) notation:
      • Example:
        0b1010; // 1 eight, 0 fours, 1 two, and 0 units
        Output: 10
  • Exponential Notation
    • Numbers can also be represented in exponential notation, which is shorthand for "multiply by 10 to the power of" (you may have heard this referred to as "scientific notation" or "standard form").
      • Examples:
        1e6; // means 1 multiplied by 10 to the power 6 (a million)
        Output: 1,000,000

        2E3; // 2 multiplied by 10^3 (two thousand)
        Output: 2,000
    • Decimal values can be created by using a negative index value:
      • Example:
        1.5e-3; // means 2.5 multiplied by 10 to the power -3 (0.001)
        Output: 0.0025
  • NaN
    • NaN is an error value that is short for "Not A Number."

  • Arrays
    • An ordered list of values
    • To create an array literal, write a pair of square brackets
      • Example:
        const myArray = [];
        Output: []
    • Can also use an array constructor function:
      • Example:
        const myArray = new Array();
        Output: []
  • Deleting Values from Arrays
    • the "delete" operator will remove any item from an array
      • Example:
        const myArray = ["Cap America", "Iron Man", "Thor", "Hulk"]
        delete avengers[3];
        Output: true
        const myArray = ["Cap America", "Iron Man", "Thor", undefined]
  • Sets
    • Introduced to the specification in ES6. A set of data structure that represents a collection of unique values, so it cannot include any duplicate values.
    • Sets offer a useful way to keep track of data without having to check if any values have been duplicated.
    • Creating Sets:
      • An empty set is created by using the new operator and Set() constructor:
        • Example:
          const list = new Set();
    • Adding Values to Sets:
      • Values can be placed into a set using the add method:
        • Example:
          list.add(1);
          Output: Set {1}
      • Multiple items can be added to the set by repeating the add() method:
        • Example:
          list.add(2).add(3).add(4);
          Output: Set {1, 2, 3, 4}
        • Example:
          const numbers = new Set([1,2,3])
          Output: Set {1, 2, 3}
      • If a string is used as the argument then each character will be added as a separate element, with any repeated characters ignored:
        • Example:
          const letters = new Set(['hello'])
          Output: Set {'h', 'e', 'l', 'l', 'o'}
      • If you want to add separate words, you need to use the add() method:
        • Example:
          const letters = new Set().add('the').add('quick').add('brown');
          Output: Set {'the', 'quick', 'brown'}
    • Weak Sets
      • When objects are added to sets, they will be stored there as long as the set exists, even if the original reference to the object is removed. The technical term for this is the object is prevented from beinggarbage-collected, which can cause amemory leak.
        • Example:
          let array = [1,2,3];
          const strong = new set().add(array);
          array - null;
          strong; // call the set
          Output: Set { [1, 2, 3]}
    • Maps
      • Mapswere another data structure introduced in the ES6 specification. They are a convenient way of keeping a list of key and value pairs, and are similar to 'hashes', or 'hash tables' or 'dictionaries' in other programming languages.
    • Creating Maps
      • An empty map object can be created using the new operator and Map() constructor:
        • Example:
          const romanNumerials = new Map();
    • Adding Entries to Maps
      • The set() method can be used to add a key and value pair to a map. The first value is the key and the second is the value:
        • Example:
          romanNumerials.set(1, 'I');
          Output: Map { 1 => 'I'}
    • Switch Statements
      • Example of If, Else Statement:
        if (number === 4) {
        console.log('You rolled a 4');
        } else if (number === 5) {
        console.log('You rolled a 5');
        } else {
        console.log('You rolled a number less than 4');
        }
      • Example of Switch statement (easier to write):
        switch (number) {
        case 4:
        console.log('You rolled a 4');
        break;
        case 5:
        console.log('You rolled a 5');
        break;
        default:
        console.log('You rolled a number less than 4');
        break;
        }

  • In JavaScript, functions are considered to be first-class objects. This means they behave in the same way as all the other primitive data types and objects in the language. They can be be assigned to variables, stored in arrays and can even be returned by another functions.
  • Function Declarations
    • Function Declation: (most common way)
      • Example:
        function hello() {
        console.log('Hello World');
        }
    • Function Expression: (assigns an anonymous functon to a variable):
      • Example:
        const goodbye = function() {
        console.log('Goodbye World');
        }
    • Function Contructors (It's not recommended to declare functions in this way as there are numerous problems associated with placing the function body inside a string):
      • Example:
        const hi = new Function('console.log("Hi World!");');
    • Arrow Functions (ES6 introduced a new syntax for declaring functions called thearrowsyntax. These make declaring functions much more succinct by using less verbose syntax):
      • Example:
        const square = x => x * x;
      • Example:
        const add = (x,y) => x + y;
      • Example:
        const hello = () => alert('Hello World!'); (If the function doesn't require any parameters, a pair of empty parentheses must go before the arrow)
      • Arrow functions have a number of advantages over other ways of declaring functions:
        • They are much less verbose than normal function declarations.
        • Single parameters don't need putting into parentheses.
        • The body of the function doesn't need placing inside a block if it's only one line.
        • The return keyword isn't required if the return statement is the only statement in the body of the function.
        • They don't bind their own value of this to the function (we'll see why this is a particularly useful property when we cover objects later in the book).

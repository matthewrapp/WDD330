  • Errors, Exceptions, and Warnings
    • Errors: Caused by when something goes wrong in the program
      • System error: There's a problem with the system or external devices with which the program is interacting.
      • Programmer error: The program contains incorrect syntax or faulty logic
      • User error: the user has entered data incorrectly
    • Exceptions: An error that produces a return value that can then be used by the program to deal with the error.
    • Warnings: Can occur if there's an error in the code that isn't enough to cause the program to crash.
  • Strict Mode
    • Strict mode produces more exceptions and warnings and prohibits the use of some deprecated features.
    • Strict mode considers coding practice that were previously accepted as just being 'poor style' as actual errors.
    • Writing in Script mode can help improve its clarity and speed.
    • Encourages a better quality of JavaScript to be written that befits a ninja programmer.
    • Recommended.
    • Simple required the following string to be added to the first line of a JavaScript file:
      • 'use strict';
    • If you want to use it only within a function, you can.
      • function strictly() {
        'use strict';
        // function code goes here
        }
  • Linting Tools
    • JS Lint
    • JS Hint
    • ES Lint
    • All can be used to test the quality of JavaScript code
    • They are designed to highlight any sloppy programming practices or syntax errors, and will complain if certain style conventions are not followed, such as how code is indented.
    • These are very useful for enforcing a programming style guide.
  • Debugging in the Browser
    • The process of finding out where bugs occur in code.
    • Lots of options for debugging JavaScript code in browser:
    • The Trusty Alert:
      • Most used form of debugging
      • alert()
      • Alert stops a program from running until OK is clicked, so it allows to effectly put breakpoints in the code that let us check the value of variables at that point to see if it's what is expected.
    • Using the Console:
      • console.log()
      • Unlike alert(), it won't stop the execution of the program
    • Debugging Tools:
      • Firefox
      • Edge
      • Chrome
      • Safari
      • Most modern browsers have a debugging tool that allows you to set breakpoints in your code that will pause it at certain points.
      • Useful command: debugger keyword
        • This will create a breakpoint in your code that will paud the execution of the code and allow you to see where the program is currently up to.
        • Example:
          • function amIOldEnough(age) {
            debugger;
            if (age < 12) {
            debugger;
            return 'No, sorry.';
            } else if (age < 18) {
            debugger;
            return 'Only if you are accompanied by an adult.';
            }
            }
    • Error Objects:
      • Can be created to the host environment when an exception occurs, or it can be created in the code using a constructor function, like:
        • const error = new Error();
        • const error = new Error('Oops, something went wrong');
      • Seven more error objects, used for specific errors: (Thrown automatically by the JavaScript engine when occurs)
        • EvalError: Not used in current ECMAScript specificion, used to identify erorrs when using global eval() function.
        • RangeError: Thrown when a number is outside an allowable range of values.
        • ReferenceError: Thrown when a reference is made to an item that doesn't exists.
        • SyntaxError: Thrown when there's an error in the code's syntax.
        • TypeError: Thrown when there's an erorr in the type of value used.
        • URLError: Thrown when there's a problem encoding or decoding the URL.
        • InternalError: Non-standard error that is thrown when an error occurs in the JavaScript engine.
      • All error objects have a number of properties:
        • name
        • message
        • stack
      • Throwing Exceptions:
        • Instead of the JavaScript automatically throwing exceptions, we can throw our own too.
          • throw new Error('Something has gone wrong!');
          • function squareRoot(number) {
            'use strict';
            if (number < 0) {
            throw new RangeError('You can't find the square root of negative numbers');
            }
            return Math.sqrt(number);
            };

            squareRoot(121);
            >> 11
            squareRoot(-1);
            >> RangeError: You can't find the square root of negative numbers
  • Test-Driven Development
    • TDD, the process of writing tests before any actual code.
    • After those tests fail, write some code to make the tests pass.
    • After, the code is refactored to make it fast, more readable, and remove and repetition.
    • Process: (The red-green-refactore cycle of TDD)
      • Write tests (that initially fail)
      • Write code to pass the tests
      • Refactor the code
      • Test refactored code
      • Write more tests more new features
    • Jest, a TDD Framework created by facebook that has gained a lot of popularity. Makes it easy to create and run tests by providing helped methods for common test assertions.
    • To use:
      • npm install -g jest
      • Check if it worked okay: jest -v >> returns the version

  • Forms
    • Made up of a form element
    • Contains form controls such as:
      • input fields
      • select menus
      • buttons
    • Traditionally, a form use to send the information to the back end server where it would be processed using a backend language like PHP or Ruby.
    • It's now possible to process the information in a form on the front end before it is sent to the server using JavaScript.
    • Each form contorl has an initial value, specified by HTML. Value can change by entering information or using a slider to increase/decrease. The value can also be changed dynamically using JS.

  • OOP
    • Often used to model representations of objects in the real world.
    • Three main components in OOP:
      • encapsulation
      • polymorphism
      • inheritance
    • Encapsulation
      • The inner workings are kept hidden inside the object and only the essential functionalities are exposed to the end user
      • Keeping all the programming logic inside an object and making methods available to implement the functionality, without the outside world needing to know how it's done.
    • Polymorphism
      • The same process cna be used for different objects
      • Various objects can share the same method but also have the ability to override shared methods with more specific implementation
    • Inheritance
      • Taking the features of one object then adding some new features
      • Taking an object that already exists and inherit all its properties and methods. We can then imporve on its functionality by adding new properties and methods
    • Classes
      • Use a class to define a blueprint for an object.
      • Objects are then created as an instance of that class, and inherit all the properties and methods of that class.
      • Classes are new to JavaScript. ES6 update.
  • Constructor Functions
    • An alternative way to create objects
    • Example of creating an object, not using Constructor syntax
      • const dice = {
        sides: 6,
        roll() {
        return Math.floor(this.sides * Math.random() + 1)
        }
        }
    • Example of creating an object, Using Constructor syntax
      • const Dice = function (sides = 6) {
        this.sides = sides;
        this.roll = function() {
        return Math.floor(this.sides * Math.random() + 1);
        }
        }
    • The keyword 'this' is used to represent the object that will be returned by the contructor function.
    • we use it to set the 'sides' property to the argument that is provided to the constructor function, or 6, if no argument is provided
    • The constructor function also adds a method called roll() , which returns a random number from 1 up to the number of sides the dice has.
    • We can now create an instance of the dice constructor function using the new operator
      • const redDice = new Dice();
        >> Dice { sides: 6, roll: [Function] }
      • Parenthesis are not required when instantiating a new object using a constructor function.
      • const redDice = new Dice;
        >> Dice { sides: 6, roll: [Function] }
      • Parenthesis are required if any default arguments need to be provided.
      • If we wanted to create another Dice object with four sides
      • const whiteDice = new Dice(4);
  • ES6 Class Declarations
    • ES6 intoduced the new class declaration syntax that does exactly the same thing as a constructor function, but looks much similar to writing a class in a class-based programming language (Java, Ruby, etc.).
    • Example of the Dice, using a class declaration
      • class Dice {
        constructor(sides = 6) {
        this.sides = sides;
        }
        roll () {
        return Math.floor(this.sides * Math.random() + 1);
        }
        }
    • By conventin, the names of constructor functions or class declarations are capitalized.
    • To create a new instance of the Dice class, the 'new' operator is again used:
      • const blueDice = new Dice(20);
        >> Dice { sides: 20 }
      • To check if blueDice is an instance of the Dice class
        • blueDice instanceOf Dice
          >> true
  • The Constructor Property
    • All objects have a contructor property that returns the constructor function that created it:
      • blueDice.constuctor
        >> [function: Dice]
    • Can use the contructor property to instantiate a copy of an object, without having to reference the actual contructor function or class declaration directly.
    • Example, if we wanted to make another copy of the redDice object, but if the name of the constructor was unknown, we could use this:
      • const greenDice = new redDice.constructor(10)
  • Static Methods
    • The static keyword can be used in class declarations to create a static method.
    • A static method is called by the class directly rather than by instances of the class
    • Example:
      • class Dice {
        constructor(sides = 6) {
        this.sides = sides;
        }
        roll() {
        return Math.floor(this.sides * Math.random() + 1);
        }
        static description() {
        return 'A way of choosing random numbers';
        }
        }
    • This method is called from the Dice class
      • Dice.description()
        >> 'A way of choosing random numbers'
    • Static numbers are not available to instances of the class
  • Prototypal Inheritance
    • JavaScript uses a prototypal inheritance model, meaning that every class has a prototype property that is shared by every instance of of the class.
    • So any properties or methods of a class's prototype can be accessed by every object instantiated by that class.
    • See example of creating a class for creating ninja turtles.
  • The Prototype Property
    • When creating a class, you would normally add any default properties and methods to this class declaration.
    • What if you want to augment the class with extrea methods and properties after the class has been created?
    • You do this by using the prototype property of the class. This is very useful when you don't have access to the class declation, but still want to add properties and methods to the class.

  • Overwriting Prototype Properties
    • Can be overwritten if inheritted from it's prototype.

  • What Should the Prototype Be Used For?
    • The prototype can be used to add any new properties and methods after the class has been declared
    • It should be used to define any properties that will remain the same for every instance of the class.
    • 'weapon' would be bad because every Turtle has a different weapon. 'food' would be good because they all love pizza.
  • The following points should be considered when using classes and prototypes to create instances:
    • Create a class declaration that deals with any initialization, shared properties and methods.
    • Any extra methods and properties that need to be augmented to the class declaration after it’s been defined can be added to the prototype. These will be added toallinstances, even those that have already been created.
    • Add any properties or methods that are individual to a particular instance can be augmented using assignment to that object (a mixin could be used to add multiple properties at once, as we’ll see later).
    • Be careful when overwriting the prototype completely ― the constructor class needs to be reset.
  • Public and Private Methods
    • By default, an objects methods are public in JavaScript.
    • Methods and properties are said to be public because they can be queried directly and changed by assignment.
    • If your objects are public facing, giving users or external services too much access to properties and methods could be a recipe for disaster.
    • We can use the concept of varable scope to keep some properties and methods private inside of a class declaration.
    • Example:
      • class Turtle {
        constructor(name, color) {
        this.name = name;
        let _color = color;
        this.setColor = color => {
        return _color = color;
        }
        this.getColor = () => _color;
        }
        }
      • In the example that follows, the Turtle() class has been modified to include a private _color property
      • The _color property is created as a variable inside the scope of the constructor function inside the class declaration
      • This makes it impossible to access outside of this scope
      • The getColor() and setColor() methods are known asgetter and setter methodsand they form a closure over this variable and provide controlled access to the property
  • The Prototype Chain
    • Can go deeper than just Object.getPrototypeOf()
    • If we do the Turtle example:
      • Object.getPrototypeOf(leo)
        >> Turtle | Class Name
      • Object.getPrototypeOf(Object.getPrototypeOf(leo))
        >> {} | Brackets
      • Object.getPrototypeOf(Object.getPrototypeOf(Object.getPrototypeOf(leo)))
        >> Null | Null, anything after this is null as well. Just keeps going backwords everytime you layer a Object.getPrototypeOf().
    • It's a chain backwards or upwards. That's why it's called the prototype chain

  • Polymorphism
    • Means that different objects can have the same method, but implement it in different ways.
    • The Object.prototype object has a toString() method that is shared by all objects.
    • Polymorphism means that objects are able to override this method with a more specific implementation.
    • Example:
      • [1,2,3].toString()
        >> '1,2,3'
      • 2..toString()
        >> '2' // Remember 2 dot operators for integers!
      • 2..toExponential();
        >> '2e+0'

  • Property Attributes and Descriptors
    • All object properties have the following attributes stored in a property descriptor:
    • Value: This is the value of the property and is undefined by default
    • Writable: This boolean value shows whether a property can be changed or not, and is false by default
    • Enumerable: this boolean value shows whether a property will show up when the object is displayed in a for in loop, and is false by default
    • Configurable: this boolean value shows whether you can delete a property or change any of its attributes, and is false by default.

  • Modular JavaScript
    • Module: self-container piece of code that provides functions and methods that can then be used in other files and by other modules.
    • Helps keep code organized in seperate, reuseable files, which improves maintainability.
    • The code in the module should have a single purpose, and group together functions with distinct functionality.
    • Example:
      • You keep any functions used for Ajax in their own module, then you could use this file for any project where Ajax is required.
    • Modules also allow public API to be exposes, while keeping the implementation hidden away inside the module
  • ES6 Modules
    • First time JavaScript allows modules
    • Things to keep in mind about modules:
      • All code in modules is always in strict mode without the need for 'use strict' and there is no way to opt out of this.
      • A module has its own global scope, so any variables created in the top-level of a module can only be accessed within that module.
      • The value of this in the top level of a module is undefined , rather than the global object.
      • You can't use HTML-style comments in modules (although this isn't very common in any JavaScript program these days).
    • Just a normal JavaScript file, but uses the keyword export to specify any values or functions that are to be made available from the module.
    • Exporting
      • export const PI = 3.1415926
      • Exporting a function:

        function variance(array) {
        ///////
        }
        function mean(array) {
        ///////
        }

        export {
        variance,
        mean
        }
      • Default Exports
        • Refer to a single variable, function or class in a module that can be imported without having to be explicitly named.
          • export default PI;
        • Using more than one default export will result in error
    • Importing
      • import { PI } from './pi.js'
